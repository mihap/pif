# Appendix to §03 Typography

## Scope

Use this appendix to turn typography tokens into a readable text system. It covers legibility, hierarchy, rhythm, and text spacing. It does not choose your brand typefaces or exact responsive breakpoints for you.

## Best Practices by Blueprint Section

### Font Stack

- Rule: Choose typefaces for screen legibility and product tone before novelty.
- Use it when: you select primary, secondary, or fallback fonts for interface copy.
- Check: the typeface stays readable in body text, labels, and dense UI rows before you optimize for personality.

### Font Weights

- Rule: Limit the working set of weights so contrast comes from clear steps instead of constant switching.
- Use it when: you define heading, body, label, and control text roles.
- Check: each weight has a distinct job and adjacent roles do not collapse into each other.

### Type Scale

- Rule: Start from body text and build a restrained scale instead of adding many near-duplicate sizes.
- Use it when: you define display, heading, body, and small-text tiers.
- Check: a reader can tell which level is stronger without needing many tiny size jumps.

### Line Height Guidelines

- Rule: Use line height to preserve rhythm and readable scanning across headings, body copy, and dense UI text.
- Use it when: you set default line-height values for text roles.
- Check: multi-line text feels open enough to scan without breaking the visual relationship between lines.

### Letter Spacing

- Rule: Adjust letter spacing only when it improves clarity for compact labels or display text.
- Use it when: a text style becomes cramped, shouty, or too loose at its intended size.
- Check: spacing changes improve legibility without making the text look like a different voice.

### Paragraph Spacing

- Rule: Separate text blocks with defined spacing rules instead of manual line breaks.
- Use it when: you lay out long-form content, help copy, or stacked labels and descriptions.
- Check: paragraphs remain readable even after copy length changes.

## DO

- Do keep the type system small enough to feel deliberate and scalable across screens.
- Do use line height and paragraph spacing to make rhythm a system choice, not a cleanup step.

## DON'T

- Don't solve hierarchy by piling on extra sizes and weights that are visually too close together.
- Don't treat text spacing like manual formatting that must be repaired every time copy changes.

## Local Decisions Required

- Decide the fallback stack rules for each approved typeface.
- Define responsive cutovers for large headings and compact utility text if your product spans very different screen classes.
