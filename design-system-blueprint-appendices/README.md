# Design System Blueprint Appendices

This folder contains self-contained sidecar appendices for `design-system-blueprint/` chapters `02` through `12`.

## How To Use These Files

- Treat the numbered files in `design-system-blueprint/` as the canonical template.
- Treat these appendix files as operational guidance that explains how to apply the template well.
- Read the appendix for the matching chapter before making product-specific choices.
- Use `Local Decisions Required` to capture choices the appendix intentionally leaves open.

## Coverage Rules

- Appendices exist for chapters `02–12` only.
- Every appendix includes `Scope`, `Best Practices by Blueprint Section`, `DO`, `DON'T`, and `Local Decisions Required`.

## Validation Rules

- Do not add appendix files under `design-system-blueprint/`.
- Do not include source maps, evidence ledgers, BP IDs, citation links, or source-corpus references inside appendix files.
- Make every rule self-contained: a reader should understand what to do, when to do it, and how to sanity-check it without external references.
